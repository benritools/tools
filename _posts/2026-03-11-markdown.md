# GithubPagesで無料公開するブログの記事をMarkDownを使って書いてみる



## 目次
* Table of Contents
{:toc}


## 1. はじめに
自分でレンタルサーバーは契約せず、サイトはGithubPagesで無料公開し、ブログ記事を追加していこうと思います。
HTMLを自分でゴリゴリと書いていくのは大変なので、 Markdown記法を使って書いていくことにしました。


## 2. そもそもMarkdownって何？
Markdown（読み方：マークダウン）記法とは、簡単な記号で文章の構造や装飾を指定できるマークアップ言語です。

*   ##​ で見出し
*   ** で強調
*   ​-​ で箇条書き

などのように、タグなどの難しい知識がなくても読みやすく綺麗なレイアウトなHTML文章が簡単に書けます。


## 3. なぜ Jekyll を採用したのか
Jekyllは、GitHubPagesに標準で組み込まれている静的サイトジェネレータでアップロードしたMarkdownファイル（.mdファイル）を自動でHTMLに変換して配信してくれます。
*   **環境構築が不要**: GitHubが標準でサポートしているため、自分で特別なツールを入れる必要がありません。
*   **Markdownとの親和性**: 書いたMarkdownファイルをリポジトリにアップロードするだけで、ブログとしていい感じの見た目になります。
*   **目次の自動生成**: 特にプラグインを追加しなくても、`{:toc} `と魔法の言葉を書くだけで目次を自動生成してくれる機能（kramdown）なんて便利なものがあります。


## 4. Markdown用のファイルを作成する
では早速、GithubPagesでMarkdownを生成（ジェネレート）してもらう為の準備を行います。

### 設定ファイル_config.ymlを作成する
GithubPagesでMarkdownファイルからHTML静的ファイルを生成する為には設定が必要となるようです。
まずは、そのための設定ファイルを_config.ymlというファイル名で作成します。

_config.ymlの例
```
url: "https://benritools.github.io" # ご自分のサイトのURL
baseurl: "/tools"                   # ご自分のリポジトリ名
theme: jekyll-theme-minimal         # 使用するテーマ　※特に凝ったデザインにしない場合は不要
permalink: /blog/:title/            # 記事のURL構造
```

作成した_config.ymlを公開リポジトリのルートディレクトリ（最上位）にアップロードします。

### 記事を書くファイルを作成する
あとは、記事用のMarkdownファイル（.mdファイル）を作成して記事の内容を書くだけです。
これを知っておくと管理が簡単になる点が３つあります。

1. 記事ファイル名のルール

Jekyllでは、記事ファイルの名前を以下の形式にしなければなりません。


2026-03-11-markdown.md　　#ファイル名の例
年-月-日-タイトル.md　　　　　#ルール


※この日付の部分は、ブログの投稿日として自動的に認識されます。


2. 記事ファイル名とURLのルール
前述の_config.xmlで設定した*記事のURL構造*が関係してきます。
もし*記事のURL構造*を何も設定しない場合、ブログ記事のURLは以下のようになります。

デフォルトの記事のURL

https://ユーザー名.github.io/リポジトリ名/年度/月/日/タイトル.html


これだとURLが長く、最後に .html がついてしまうのがなんだか少し気になりますよね...

一方、*記事のURL構造*に

permalink: /blog/:title/            # 記事のURL構造

を指定する場合、permalinkを指定した記事のURLはこんな風になります。


https://ユーザー名.github.io/リポジトリ名/タイトル/

どうですか？とてもすっきりして覚えやすいURLになりました。
シンプルで美しいURLはSNSでシェアされた時や検索結果に表示された時の印象がぐっと良くなります！


### 作成した記事ファイルをアップロードする

記事用のMarkdownファイルを作成したら、Githubリポジトリの下記のディレクトリにファイルをアップロードします。


例）リポジトリ/_posts/2026-03-11-markdown.md

ファイルをGithubにアップロードしてコミットすると、**Actions**が動き、**pages build and deployment**が自動で実行されてコンテンツが生成されます。

:::note warn
もしも正しく反映されない場合、記事をアップロードするディレクトリ（_post）や、設定ファイル（_config.xml）のファイル名などに間違いがないかご確認ください。

ちなみに私は、何度アップロードしても全然反映されずに悩んでいたところ、ディレクトリ名を間違って_postにしていました...
:::
