# GithubPagesで無料公開するブログをMarkDownで記事を書いてみる



## 目次

## 1. はじめに
自分でレンタルサーバーは契約せず、サイトはGithubPagesで無料公開した。そのサイトにブログ記事を追加していこうと思う。
HTMLを自分でゴリゴリと書いていくのは大変なので、 Markdown記法を使って書いていくことにした。


## 2. そもそもMarkdownって何？
Markdown（読み方：マークダウン）記法とは、簡単な記号で文章の構造や装飾を指定できるマークアップ言語です。

*   `​##`​ で見出し

*   `**`​ で強調

*   `​-`​ で箇条書き

つまり、タグなどの難しい知識がなくても読みやすく綺麗なレイアウトなHTML文章が簡単に書けます。


## 3. なぜ Jekyll を採用したのか
Jekyllは、GitHubPagesに標準で組み込まれている静的サイトジェネレータでアップロードしたMarkdownファイル（.mdファイル）を自動でHTMLに変換して配信してくれます。
*   **環境構築が不要**: GitHubが標準でサポートしているため、自分で特別なツールを入れる必要がありません。

*   **Markdownとの親和性**: 書いたMarkdownファイルをリポジトリにアップロードするだけで、ブログとしていい感じの見た目になります。
  
*   **目次の自動生成**: 特にプラグインを追加しなくても、`{:toc} `と魔法の言葉を書くだけで目次を自動生成してくれる機能（kramdown）なんて便利なものがあります。
*   


## 4. Markdown用のファイルを作成する
では早速、GithubPagesでMarkdownを生成（ジェネレート）してもらう為の準備を行う。

### 設定ファイル_config.ymlを作成する
GithubPagesでMarkdownファイルからHTML静的ファイルを生成する為には設定が必要となるようだ。
まずは、そのための設定ファイルを_config.ymlを作成する。

### _config.ymlの設定例

GitHub Pagesの設定ファイルである `_config.yml` を以下のように編集します。

```_config.yml
url: "https://benritools.github.io" # ご自分のサイトのURL
baseurl: "/tools"                   # ご自分のリポジトリ名
theme: jekyll-theme-minimal         # 使用するテーマ　※特に凝ったデザインにしない場合は不要
permalink: /blog/:title/            # 記事のURL構造
```

作成した_config.ymlを公開リポジトリのルートディレクトリ（最上位）にアップロードする。
