## 1. はじめに
前回、[GithubPagesでブログ記事をJekyllを使ってMarkDownで書いてみる]({{ '/blog/markdown/' | relative_url }}){:target="_blank"}でページを作成してみました。
これからどんどんページが増えていくことを考えると、ヘッダーやフッターなどやcssなど共通になる部分を毎回書くのは面倒で、メンテナンス性も悪いです。
そこで、各ページで共通化できる部分をテンプレートとして定義することにしました。
この記事では、Jekyllを使ってMarkDownで静的ページを作る場合に共通で読み込むテンプレートファイルの作り方を紹介しています。

## 2. 共通化したいところって？
ページを複数作成したときに、共通化した方がいい部分を考えてみました。

*   画面上部の`ヘッダー` に、タイトルを共通で出したい。
*   画面下部の`フッター` に、プライバシーポリシーやお問い合わせページへのリンクを貼りたい。
*   ​ページの背景色やフォントなどのデザインを統一したい。

などのように、全ページで必要になる部分のデザイン・形式や文章をあらかじめ用意しておき、全てのページで統一されたデザインや見た目を実現したいと思います。

## 3. ブログサイトの共通テンプレートを作成する
**サイト全体のベース**となるHTML構造を定義します。
つまり、`<html><head><body>`などに当たる部分で共通して使いたい部分を定義します。例えば、

*   `<html>`で全てのページで日本語を使うことを定義する。

*   `<head>`でcssは同じものを読み込んでデザインを統一する。

*   `<body>`でヘッダーを`<header>`に定義して、フッターを`<footer>`に定義して同じものを読み込む

など、全ページで共通で使う部分を書きます。

_layouts/default.htmlの例
{% raw %}
```html
<!DOCTYPE html>
<html lang="ja">
<head>
<meta charset="UTF-8">
<title>{{ page.title }}</title>
</head>

<body>

<header>
<a href="/tools/">BenriTools</a>
</header>

<main>
{{ content }}
</main>

</body>
</html>
```
`{{ content }}` の部分には、ブログ記事の内容が表示されます。
default.htmlファイルを作成したら、Githubリポジトリのディレクトリにファイルをアップロードします。
{% endraw %}

## 4. ブログ記事の共通テンプレートファイルを作成する

[_layouts/post.html]
{% raw %}
```
---
layout: default     #ここで親のテンプレートを指定しています
---

<article>

<h1>{{ page.title }}</h1>

<p>{{ page.date | date: "%Y-%m-%d" }}</p>

{{ content }}

</article>
```
`{{ content }}` の部分には、ブログ記事の内容が表示されます。
post.htmlファイルを作成したら、Githubリポジトリのディレクトリにファイルをアップロードします。
{% endraw %}

## 5. 各ブログ記事ページで使用するテンプレートファイルを指定する
各ブログ記事ページで共通テンプレートファイルを使用する為には設定が必要となります。
各記事ページのlayout項目にどのテンプレートを使うかを指定します。

[各記事ページの.mdファイル]
```
---
layout: post     #post.htmlのテンプレートを使う指定
title: "GithubPagesでMarkdownブログを書く"
date: 2026-03-11
---

# GithubPagesでMarkdownブログを書く

本文
```

各記事ページの.mdファイルを更新します。すると、以下の順番でテンプレートファイルが読み込まれてブログ記事ページが表示されます。



[参考）ディレクトリ例]
```
tools
 ├ _layouts
 │   ├ default.html
 │   └ post.html
 ├ _posts
 │   └ 2026-03-11-markdown.md
 └ index.html
```

### 記事を書くファイルを作成する
あとは、記事用のMarkdownファイル（.mdファイル）を作成して記事の内容を書くだけです。
これを知っておくと管理が簡単になる点が2つあります。

#### 1. 記事ファイル名のルール

Jekyllでは、記事ファイルの名前を以下の形式にしなければなりません。

```
YYYY-MM-DD-title.md
```
YYYY-MM-DD部分は更新日、titleの部分は記事のタイトルを半角英数で指定します。

※日付の部分は、ブログの投稿日として自動的に認識されます。

[例]
```
2026-03-11-markdown.md
```


#### 2. 記事ファイル名とURLのルール
前述の_config.xmlで設定した*記事のURL構造*が関係してきます。
もし*記事のURL構造*を何も設定しない場合、ブログ記事ページのURLは以下のようになります。

[デフォルトの場合の記事ページのURL]
```
https://ユーザー名.github.io/リポジトリ名/年度/月/日/タイトル.html
```

これだとURLがとても長く、最後に .html がついてしまうのがなんだか少し気になりますよね...

一方、*記事のURL構造*に
```
permalink: /blog/:title/
```
を指定した場合、permalinkを指定した記事のURLはこんな風になります。

```
https://ユーザー名.github.io/リポジトリ名/タイトル/

例）
https://benritools.github.io/tools/blog/markdown/
```

どうですか？とてもすっきりして覚えやすいURLになりました。
シンプルで美しいURLはSNSでシェアされた時や検索結果に表示された時の印象がぐっと良くなります！


### 作成した記事ファイルをアップロードする

記事用のMarkdownファイルを作成したら、Githubリポジトリの下記のディレクトリにファイルをアップロードします。


[ディレクトリ例]
```
リポジトリ/_posts/2026-03-11-markdown.md
```

## 5. Markdownファイルから記事ページを生成する
アップロードしたMarkdownで書いた記事ページからHTMLを生成する時にすることは、なんと何もありません！

ファイルをGithubにコミットすると、Actionsが**自動的に**動き、pages build and deploymentが実行されて**記事が自動生成**されます。

ややこしい操作もなく、自動ですべてやってくれるので本当に簡単で驚きです😱

> **Warning**
> もしも正しく反映されない場合は、記事をアップロードしたディレクトリ名（`_posts`）や、設定ファイル（`_config.yml`）のファイル名などに間違いがないかご確認ください。
> ちなみに私は、何度アップロードしても全然反映されずに悩んでいたところ、ディレクトリ名を間違って `_post` にしていました...😂
